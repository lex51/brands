Дана таблица c данными о товарах и их конкурентах (data.xlsx). В поле “Конкуренты WB” перечислены артикулы аналогичных товаров конкурентов через запятую. 

Задание:
 1. Положить данные в свою опубликованную базу данных PostgreSQL;
 2. Произвести парсинг конечных цен конкурентов (после всех скидок) на WB;
 3. В зависимости от выбранной стратегии ценообразования для товара (поле “Стратегия ценообразования”), установить цену в поле “Новая цена”;
 4. Установить значение false в поле “В наличии у конкурентов”, если данный товар не в наличии у всех конкурентов (у товаров не в наличии на сайте не отображается цена). Установить значение true в в поле “В наличии у конкурентов”, если данный товар в наличии у конкурентов;
 5. Заполнить поле “Дата парсинга” датой и временем парсинга конкурентов;
 6. Написать API и мини-документацию для получения полей из Вашей базы данных.

Примечания
 - В случае, если ни один из конкурентов не имеет наличие на данный момент, установить в поле “Новая цена” значение NULL.
 - В таблице заполнена первая строка для демонстрации желаемого вида данных. Необходимо удалить данные из ячеек “Дата парсинга”, “В наличии у конкурентов”, “Новая цена”.
 - В названиях полей также для удобства написан тип данных.
 - Товары на Wildberries доступны по ссылке https://www.wildberries.ru/catalog/артикул/detail.aspx?targetUrl=SP. Артикул необходимо заменить на реальный.


Объяснение стратегий ценообразования:
EDLP - установить цену на 1% меньше, чем у самого дешевого конкурента.
AVERAGE - установить среднюю арифметическую цену от конкурентов.


Результат:
Мы хотим получить от Вас API методы для получения данных по полям: “Новая цена”, “В наличии у конкурентов”, “Дата парсинга”. В API запросе будет передаваться артикул товара из столбца “Артикул”.
